# 截图导入故障分析与美国 VPS 部署结论

更新时间：2026-07-10

## 结论

股票代码、数量、市值、期权 Delta 等字段的截图识别不是难题；`glm-4.6v-flash` 与 `glm-4v-flash` 都是官方支持图片输入的模型。此前失败主要由两类独立问题造成：

1. GitHub Pages 的手机浏览器直接请求 AI API，长时间图片请求在移动网络上容易超时；这不是截图压缩或域名缺失造成的。
2. 截图中显示“您的账户已达到速率限制”。这是智谱按**账户和模型**施加的并发/频率限制；换域名、VPN 或服务器不能绕过，只能减少并发、等待冷却，或使用有可用额度/权益的 Key。

美国 VPS 将作为最终部署服务器；此前阿里云已验证可在约 0.17 秒连接到智谱 API 并取得鉴权响应。最终链路改为：

`手机浏览器 → 美国 VPS 同源网关 → 智谱 / Kimi API`

手机不再对 AI API 建立跨境/跨运营商的长连接，且网关会按 API Key 串行处理请求，避免“连接测试”和“截图解析”同时在途而触发账号并发限流。

## 已验证的事实

| 检查 | 结果 | 含义 |
| --- | --- | --- |
| 服务器到智谱 `/chat/completions` | 约 0.17 秒返回 401（故意使用无效 Key） | 服务器网络、DNS、TLS 与 API 路由正常 |
| 网页的智谱连接测试 | 返回“账户已达到速率限制” | Key 已到达智谱，失败点是账号/模型限流，不是网页无法联网 |
| `glm-4.6v-flash` 官方能力 | 支持图像、文本、Base64 图片 | 可用于券商截图结构化提取 |
| GitHub Pages 直连方式 | 手机出现 90 秒超时 | 应由服务器转发取代 |

## 本次修正

- 新增美国 VPS 网关：`/api/zhipu/chat/completions`、`/api/kimi/chat/completions`、`/api/quotes`、`/api/health`。
- 最终网页地址为 `http://67.215.255.196:8788/`；它使用独立的 `/opt/portfolio-tracker-*` 目录、独立 PM2 `portfolio-tracker` 进程和独立 8788 端口。既有 StockPulse/OpenCoolAI 进程、Nginx、80/443 和 3000 均未修改。
- 网关不保存 API Key；Key 只在浏览器本地保存，并在请求时转发。网关内存只保存 Key 的 SHA-256 摘要用于串行排队。
- 智谱截图请求显式关闭深度思考，输出上限从 8000 降至 3500，减少不必要的处理时间。
- 网关上游超时设为 180 秒，Nginx 同步放宽到 190 秒；前端默认使用同源阿里云网关。
- 部署版本自动启用免费 Yahoo 日线/收盘价行情代理；网页在北京时间早上 7 点后首次打开时刷新一次，不做实时盯盘。
- 当 AI 返回 `429`、`1302` 或“速率限制”时，页面会明确提示这是账号限制，不再误导为 VPN、域名或截图问题。

## 域名是否必须

**功能上不必须。** 可以先通过美国 VPS 的 `http://IP:8788/` 使用，无需购买域名。

**安全和正式对外使用时建议必须有。** 公网 IP 不能方便地获得长期可信的 HTTPS 证书；HTTP 会让 API Key 在网络中缺少传输加密。后续若购买域名并完成中国大陆服务器所需备案，即可在同一台服务器上配置 HTTPS，网站代码和网关代码无需重写。

目前该服务是个人自用、服务器地址访问；请不要把链接公开分享，也不要在不可信 Wi-Fi 下输入 API Key。迁移到域名 HTTPS 后可去掉这个限制。

## 如何验证实际截图导入

1. 打开阿里云部署网址。
2. 在“设置”确认显示“已启用阿里云中转”，填入智谱 Key，点“保存设置”。
3. **不要连续点连接测试**；等待约一分钟后，只点一次“测试智谱 GLM 连接”。
4. 测试成功后到“持仓”上传一张完整持仓截图，点“解析并导入”。
5. 成功后自动进入总览；若缺期权详情或现金页，仍先显示已识别持仓，并在总览列出需补充项。

如果第 3 步仍返回“速率限制”，网关已经证明链路正常。需要在智谱开放平台查看该 Key 对 `glm-4.6v-flash` 的速率权益，等待冷却，或更换有可用额度的 Key；不应通过连续重试解决。
